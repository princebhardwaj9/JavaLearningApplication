    Before starting this tutorial, did you know about packages?

  • Packages are nothing more than the way we organize files into different directories according to their functionality, usability as well as category they should belong to.
  • Packages are the collection of similar types of classes & interface.
  • No, class can exist without any package in Java. If you have not given any package making command in .java file then compiler will create one default package in the same folder and put .class file into that folder, that default package doesnot have any name.
  • Example: files in java.io package do something related to I/O but files in java.net give us the way to deal with Network.

    • Note: Package names are written in all lower case to avoid conflict with the names of Classes on Interfaces.

To create a class inside a package, you should declare the package name as the first statement of your program.

Import

  • So import is the keyword in Java.
  • It is used to import classes of other packages.
  • import keyword is used to import built-in and user-defined packages into your java source file so that your class can refer to a class that is in another package by directly using its name.
  • Importing a Package Member

    To import a specific member of the package we need to mention the name of the member like import java.util.Scanner as if we need Scanner class only from util package we import only Scanner class.

  • Importing Entire Package

    To import all the types contained in a particular package, use the import statement with the asterisk(*) wild character, like if we want to import all member of util package, we need to write like import java.util.*;.

All import statements should be written above the declaration of the class.
