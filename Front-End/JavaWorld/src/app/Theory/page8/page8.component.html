Yes! you are right,
Java is not 100% Object Oriented Programming Language because of concept of these primitive data types like int, char, long, short etc.

A Object Oriented Programming Language is what when all the data is present in the form of object but because of these primitive data types it would not be possible but Java provide a solution for this & define a concept of wrapper classes.

Wrapper Classes

  • A solution to this problem, Java allows you to includes the primitives in the family of objects by using wrapper classes.
  • A Wrapper class is a class whose object wraps or contains a primitive data types. In other words, we can wrap a primitive value into a wrapper class object.
  • Advantage of the wrapper class is that it follows the Object Oriented approach as well as then the manipulation of data is more easy & efficient by use of functions.
  • There is a wrapper class for every primitive data type in Java.
  • This class encapsulates the single value of primitive data type.
  • For instance the wrapper class for int is Integer, for float is Float, char is Character, and so on.
  • As you see we use class naming convention to name the wrapper classes.
Primitive Data Type Wrapper Class
boolean Boolean
byte Byte
char Character
short Short
int Integer
long Long
float Float
double Double
