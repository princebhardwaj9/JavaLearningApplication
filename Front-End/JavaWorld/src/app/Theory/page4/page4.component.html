Variables

"Variables are just a name given to a memory location and that memory location is used to hold the data or values."


Java defines the following kinds of variables:

  • Instance Variables (Non-Static Fields): Objects store their individual states in "non-static fields", the fields declared without static keyword. They are also known as instance variables because their values are unique to each object OR to each instance of a class.

  • Class Variables (Static-Fields): They are the fields/variables declared with static modifier, which tells the compiler that there is exactly one copy of this variable in existence, regardless of how many times the class has been instantiated(regardless of how many times the object of that class has been made, in other words).

  • Local Variables: Like I tell you that a object store its state in fields, a method(function) will often store its temporary state in local variables. Local variables are not accessible out from the method block, they are only visible to the methods in which they are declared.

  • Arguments: The important thing to remember is that parameters are always classified as "Variables" not "Fields". You will learn about later in the tutorial.

Naming

Everey programming language has its own set of rules and conventions for kind of names that you're allowed to use. Rules & conventions for naming are as follows:

  • Variables name are case-sensitive (int myVar & int myvAr both are different) it must be start with a letter, $(Dollar), _(Underscore).

    Valid: myVar, $myVar, _myVar

    Invalid: 8myVar, #myVar

  • Name you choose of the variable must not be a "KEYWORD or RESERVED WORD".
  • Use the Camel-Case for naming the variable.

    Like: yearlyInterest, firstName, getData
